About the Project

This java program is a wrapper for creating and pushing a project from your local machine to Github. This project was created as a project for the course CSC-109 (Intro to Git) at Quinnipiac Univeristy.

Installing the Project

Required Software

This project requires that you have git and Java 17 installed. While it is possible that the project may run on older versions of Java, we developed this program using Java 17.

Setting Up the Project

First, you will need to download the project from git. To do this, ensure that git is installed on your computer, and execute the following code:

git clone https://github.com/LittleTealeaf/CSC-109-GitApp

Once you've cloned the repository, open it up with your favorite Java IDE. For this project, we used Visual Studio Code, but IDEs such as IntelliJ or Eclipse work perfectly fine.

In this project, we used two external libraries that were provided to use. These libraries wrapped the Github API and git functions to expose an easy API that we could use. You can view the repositories for the libraries used as follows:

  • GithubAPIClient by Alex Thimineur
  • GitSubprocessClient by Alex Thimineur

For ease of use, these libraries have been pre-downloaded and included in our repository, located in the libs/ directory.

However, you will still need to tell your IDE that you wish to use those packages in your program. Here are links to articles instructing you on how to do so with the IDEs mentioned above:

  • Visual Studio Code: Manage dependencies for unamanged folders
  • IntelliJ: Defining a Project Libraries
  • Eclipse: Adding User Libraries to Eclipse

Once you've properly set up both libraries (found in libs/), you are ready to start the program! You will find the main method in src/App.java. Using your IDE, execute the static void main(String[] args) {...} method to launch the application

Using the Application

Once you've launched the application, you will be greeted with the following ui. Given this is a prototype, of course the UI isn't all that great.

Github API Token: you will need to provide the application a Github API Token that has permissions to access your account. You can read instructions on creating and getting a Github API Token from Github Documentation

Github Username: Please enter your Github Username here, so we can make sure that the repositories go in the right space!

Repository Path: Please enter the file location of project workspace that you would like to convert to a Github Repository. This must be the full absolute path.

Repository Name: What do you want your repository to be called on Github? This is where you can name it! The name should adhere to the following restrictions

  • All characters must either be ASCII Alphanumeric, a dash (-), an underscore (_), or a period (.)
  • Cannot contain only periods (.)

Repository Description: What is this repository about? Whatever you put in here will be auto-filled into the README.md file, as well as the repository description on Github

Once you've completed all the previous steps, you're ready to hit Submit, and the application will turn your previously dull project into a git repository hosted on GitHub!
